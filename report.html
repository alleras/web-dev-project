Web Development Final Assignment

After seeing how the rental market has been crazy for so long after the pandemic, I've been more and more interested in becoming a homeowner. This sparked the idea to make a website where I can project a fictitious non-profit that helps people along this very stressful journey. I could see an organization like this offering services, tools and information, all in one place. So that's exactly what I show on my site.

1. Seeing the amount of information that is out there regarding buying a house, and how complicated the process is. This mainly inspired the thought that a WebSite where all that was condensed was a good option for my assignment.

2. I was very much inspired, oddly, by Windows 11's design language. I loved implementing the slightly rounded corners and slight accents that are present throughout that Operating System. Paying attention to design is one of my goals when working on a website.

3. Other blogs I have worked on, and Handlebars.js. The idea to make a blog came very quickly for me when watching the videos for the class pertaining Handlebars. I thought it was a very neat way to template things that resembled the way Wordpress works (you design one template, and then the system loops through the data and shows as many as needed).

1. I tried to make good use of good contrast ratios for colors in links and buttons. The intention of it should be clear enough that the user knows it's an action that needs to or can be performed, and the text should be legible.

2. The site uses styles that make it obvious that an object is being interacted with, even if the mouse is not involved in the action. An example of this is the input boxes in the About site.

3. I added accessibility options such as bigger text (for people with impaired sight) and underlining links, for users that need help identifying if a button or text will take them somewhere or not.

1. I considered user interactions with elements such as the carousel and the image collage to be a great opportunity for usability. These objects can be interacted with, which enhances the user experience or the amount of information that the website can show.

2. The calculator page was also a big usability item in my list. It allows the user to take something more than just general information out of it, instead providing something suited to their needs.

3. A smaller, albeit neat, usability component is the "Scroll to Top" button, which makes navigation easier if users do not want to be abusing their mouses when scrolling!

1. I had to relearn how to make a carousel. I learned how to make them in the web many years ago, and most of those techniques were already outdated. So I found a tutorial online that walks you through the code that needs to be added to the HTML, CSS and JS. Then, I made sense of it by seeing how it worked and what it did. After that I modified it to adapt and extend its functionality for my site.

2. I found out that styling can be done a lot easier for the mobile if it's developed first, and then things are adapted to the desktop. It looks like working with the constraints of the smaller screens makes us code in a smarter and cleaner way.

3. I had to make some research on several JavaScript Browser functions. Fetch, for instance, was a function that I had already used, but I didn't know it could easily be used to load text in a file retrieved locally. This became the basis for my templating system. Another thing I researched more in depth was the event listeners in JavaScript, as a consequence of me using templates.

My usage of Handlebars.js as a templating system allowed me to not have to rewrite the same code on each site that I published, instead I was able to make .hbs files with the templates for the Header and Footer. Then, I turned these files into Handlebars Partials which I loaded with JavaScript and dynamically incorporated into the code, along with the content of each page. Initially, I wanted to make templates as separate files to keep my content clean, but Handlebars (browser) does not offer that funcionality. It is, instead, available on server-based versions of it. I was able to solve this problem by using JavaScript Fetch() to get the content of the templates and register them as partials. I go more in depth in the comments of the /js/util.js file.

Another element I feel worked very well was the overall design. I was able to find a simple yet very cohesive design language for this specific WebSite, which made my job a lot easier as I didn't have to style an excessive amount of objects.

In general, I believe I could handle the responsiveness of the site better. This could be solved by becoming more detailed in my media breakpoints and tailoring for each viewport. There are some elements that currently resize poorly when they are in between common sizes. I don't expect that to be a big problem, but the perfectionist in me would like to see solved.
